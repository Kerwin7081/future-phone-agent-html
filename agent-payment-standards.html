Core Thesis

Agent 支付不是单一赛道，而是支付协议、授权协议、钱包账户、结算网络、身份风控和责任归属的系统性重构。

Stripe 有一套，Coinbase 有一套，但整个市场远不止这两家。Stripe / OpenAI ACP 更偏 AI 购物与商户结账；Coinbase x402 更偏机器对机器支付与 API 微支付；Google AP2 更偏授权委托；Visa TAP 和 Mastercard Agent Pay 更偏可信 Agent 身份与卡网络；PayPal 更偏钱包入口和争议处理；Circle 更偏稳定币结算资产和机构网络。

Framework

一、Agent 支付的五层拆解

Agent 支付不能简单理解成“AI 帮你刷卡”。真正的问题是，当 Agent 从信息助手变成经济执行主体之后，支付系统必须回答五个问题。

第一层：Agent 购物与下单协议

这一层解决的是：AI Agent 如何代表用户向商户下单。Agent 需要知道商品、价格、库存、税费、配送、退货政策，也需要把用户确认过的购买意图传递给商户后端。

这正是 Stripe / OpenAI ACP 的主要位置。它不是稳定币协议，也不是链上支付协议，而是 AI App 与商户系统之间的交易语言。

第二层：支付授权协议

这一层解决的是：用户如何授权 Agent 花钱。授权不能只是“我同意你买东西”这么粗糙。未来需要明确金额上限、商户范围、商品类别、有效时间、是否可重复使用、是否需要二次确认、是否绑定特定上下文。

Google AP2 的核心价值就在这里。它试图把 Agent 支付中的用户意图和授权委托标准化。

第三层：钱包与资金账户

Agent 最终要从某个资金账户里扣钱。这个账户可以是银行卡、PayPal 钱包、Stripe Link、企业资金账户、稳定币钱包，也可以是嵌入云平台或企业软件里的预算账户。

这一层是 Stripe、PayPal、Coinbase、MetaMask、Bridge、Privy 等生态的交叉地带。

第四层：结算网络

支付前端可以很像，但底层结算网络可能完全不同。消费者购物可能走 Visa 或 Mastercard，平台收款可能走 Stripe，跨境出款可能走 PayPal 或银行网络，机器微支付可能走 USDC，机构后台结算可能走 Circle CPN。

这也是为什么 Agent 支付不会只有一个赢家。前端协议、钱包账户和底层结算网络会长期并存。

第五层：身份、风控、审计与责任归属

当大量 Agent 自动访问商户、API、数据源和金融服务时，商户必须知道对方是可信 Agent 还是恶意机器人。支付网络也必须知道这笔交易是否获得用户授权，是否存在 prompt injection、重复支付、越权支付或欺诈风险。

Visa TAP、Mastercard Agent Pay 和 Google AP2 都在处理这一层问题，只是切入角度不同。

Stripe / OpenAI

二、Stripe / OpenAI ACP：主打 AI 购物与商户结账

Stripe 和 OpenAI 联合推出的是 Agentic Commerce Protocol，简称 ACP。

ACP 的核心不是链上支付，也不是稳定币本身，而是让 AI Agent 和商户后端能够完成一次标准化购买流程。

在这个模型里，用户可以在 ChatGPT 或其他 AI App 中发现商品，Agent 帮用户整理选择、确认订单，最后通过商户后端完成购买。商户仍然是 merchant of record，继续负责履约、退货、客服、库存、税费和客户关系。

这点非常重要。ACP 并不是要把商户关系全部交给 AI 平台，而是让 AI 平台成为新的购物入口，同时让商户保留交易主体地位。

Stripe 的优势

Stripe 的优势在于它本来就深度嵌入电商、SaaS、平台经济和商户支付系统。它有 Checkout、Payment Links、Billing、Connect、Treasury、Link 钱包和成熟的开发者 API。

因此，Stripe 做 Agent 支付的逻辑不是从零创建一个新支付网络，而是把已有的商户支付能力变成 Agent 可以调用的标准接口。

如果一个商户已经使用 Stripe，那么接入 ACP 或 AI commerce 的边际成本可能较低。如果商户没有使用 Stripe，也可以通过 Delegated Payments 或共享支付凭证的方式参与。

Agentic Commerce Suite 的意义

Stripe 后续推出的 Agentic Commerce Suite，进一步强化了它在 AI 商户支付编排层的位置。用户可以授权 Agent 代表自己付款，真实支付信息不暴露给 Agent，每个任务可以发行一次性卡或一次性支付凭证，每笔关键支付仍可要求用户批准。

这套逻辑的重点是：Agent 可以执行支付任务，但支付凭证、风控、用户授权和商户结算仍由 Stripe 等受监管基础设施控制。

所以，Stripe 的路线可以概括为：用现有商户支付网络、Link 钱包、一次性支付凭证和 ACP，成为 AI 购物场景里的支付编排层。

Stripe 更像是 AI 商户支付入口。它解决的是“Agent 如何帮用户买东西”。
Coinbase

三、Coinbase x402：主打机器对机器支付、API 付费和稳定币微支付

Coinbase 的路线和 Stripe 完全不同。它的核心是 x402。

x402 的思想来自 HTTP 402 Payment Required。传统互联网长期没有真正实现“访问资源前自动付款”的开放标准，但 Agent 时代重新激活了这个需求。

一个 Agent 可能要调用搜索 API、数据源、模型推理服务、MCP server、网页内容、金融数据、代码工具或另一个 Agent 的服务。如果每次调用都依赖人工支付确认，Agent 经济就无法高频运转。

x402 的典型流程

典型逻辑是：Agent 访问某个资源，服务端返回“需要支付”，Agent 钱包根据预设授权自动完成稳定币支付，支付完成后获得资源访问权。

这套逻辑不是面向“人类用户买商品”，而是面向“机器访问资源时自动付费”。

因此，x402 更适合 API 调用、MCP 工具调用、数据抓取、AI 推理服务、模型调用、网页内容付费、机器间结算、微服务付费和按次计费。

Coinbase 的战略位置

Coinbase 的优势是 Base、USDC、Agent Wallets 和开发者生态。它可以让 Agent 拥有可编程钱包，并通过稳定币进行微支付。

如果未来大量 Agent 调用工具、模型、API、数据源都需要按次支付，那么 Coinbase 的角色就不只是加密货币交易所，而是 Agent 经济里的钱包和支付协议基础设施。

AWS Bedrock AgentCore Payments 把 Coinbase 和 Stripe 纳入企业 Agent 支付框架，也说明 x402 这类协议已经开始进入云平台和企业 Agent 开发层。

Coinbase 更像是 Agent 原生机器支付协议押注。它解决的是“机器调用资源如何自动付费”。
Google AP2

四、Google AP2：主打开放式 Agent 支付授权标准

Google 的路线是 Agent Payments Protocol，简称 AP2。

AP2 的定位更偏标准层和授权层。它关注的不是某一个钱包，也不是某一种结算资产，而是 Agent-led payments 里最关键的授权问题。

未来 Agent 不可能每一次执行任务都让用户重新输入银行卡，也不能只依赖一次模糊授权。用户需要提前设定可验证的 mandate，即授权委托。

Mandate 的核心

一个合格的 mandate 至少需要回答这些问题：

  • Agent 可以为谁付款？
  • 可以买什么类别的商品或服务？
  • 最高金额是多少？
  • 可以在哪些商户或平台使用？
  • 授权什么时候过期？
  • 是否允许重复支付？
  • 是否必须绑定某个任务上下文？
  • 是否需要用户最终确认？

如果没有这类授权标准，Agent 支付很容易出现越权、重复支付、被恶意提示词诱导、上下文漂移和责任不清。

AP2 的战略意义

AP2 可以理解为 Agent 支付里的授权中间层。它可以与 A2A、MCP、钱包、卡网络、稳定币和商户系统组合使用。

Google 选择做这一层很合理。它不一定要控制每一种支付方式，但如果它能定义 Agent 支付授权的通用语言，就可能影响整个生态的信任基础。

AP2 解决的是“Agent 是否有权花这笔钱”。
Visa / Mastercard

五、Visa TAP 与 Mastercard Agent Pay：卡组织不会轻易被稳定币绕开

很多人一看到稳定币支付，就会认为 Visa 和 Mastercard 会被颠覆。这个判断过于简单。

卡组织真正强的不是一张塑料卡，而是身份、授权、风控、tokenization、争议处理、全球商户接受网络、发卡行和收单行关系，以及几十年积累的合规与责任划分机制。

Visa Trusted Agent Protocol

Visa TAP 关注的是可信 Agent 身份。未来大量 AI Agent 会访问商户网站，商户必须知道这是合法购买代理，还是爬虫、欺诈机器人、攻击流量或异常自动化脚本。

TAP 的目标是让被批准的 Agent 能够安全传递关键信息给商户，帮助商户识别真实的 commerce intent。

Mastercard Agent Pay

Mastercard Agent Pay 则更强调把 Agent 纳入现有卡网络基础设施。它提出 Know Your Agent，通过注册 Agent、网络 token、身份治理和可追踪交易，让 Agent 可以在卡网络中安全付款。

Visa 和 Mastercard 的共同目标很清楚：不是让 Agent 绕开卡网络，而是让 Agent 成为卡网络上的新型支付主体。

卡网络的中短期优势

在消费者购物场景里，卡网络仍然有明显优势。原因是消费者需要退款、争议处理、信用支付、欺诈保护和商户接受度。稳定币在这些环节还没有完全替代卡网络。

长期风险在于，如果大量 Agent 支付迁移到 API 微支付、机器间结算和 B2B 稳定币清算，卡网络的部分利润池会受到挤压。

Visa 和 Mastercard 的策略是把 Agent 纳入卡网络，而不是让 Agent 绕开卡网络。
PayPal

六、PayPal：钱包入口、商户网络和争议处理层

PayPal 没有像 Coinbase 那样推出一个完全独立的机器支付协议，也没有像 Google 那样主导开放授权标准。它更像是 Agentic Commerce 的钱包入口和消费者保护层。

PayPal 的优势是用户已经有钱包，商户已经接入 PayPal，消费者也熟悉 PayPal 的买卖双方保护、退款、物流追踪和争议处理机制。

Agent 支付里的争议处理价值

Agent 支付普及后，争议处理会变得更重要。

如果 Agent 买错东西，谁负责？如果 Agent 被诱导购买不需要的商品，能不能退款？如果 Agent 重复下单，如何申诉？如果商户履约异常，用户找谁处理？

这些问题正是 PayPal 的传统强项。

所以 PayPal 的路线可以概括为：把现有钱包、商户网络、买卖双方保护和争议处理能力接入 Agent 购物时代。

PayPal 的价值不只是支付按钮，而是用户愿意授权给 Agent 的可信钱包和售后责任层。
Circle

七、Circle：稳定币结算网络和 Agent 微支付资金层

Circle 不一定是 Agent 支付协议的直接制定者，但它是非常关键的结算层玩家。

Agent 支付不仅包括买鞋、订酒店、叫车这些传统消费，也包括 API 每次调用、MCP 工具调用、数据源访问、模型推理按 token 结算、Agent 给另一个 Agent 付服务费。

这些场景很多都具有小额、高频、全球化、自动化、可编程和 24/7 的特征。传统卡网络和银行体系在这些场景里成本较高、结算时间较长、跨境摩擦较大。

USDC 的结算价值

USDC 是 Agent 支付最重要的候选结算资产之一。它具备美元计价、链上可编程、全球流通、全天候结算和开发者生态优势。

Circle 的 CPN、Managed Payments、Gateway 和 Nanopayments 共同构成一套更完整的稳定币结算基础设施。

其中，CPN 更偏机构级支付网络，Managed Payments 更偏托管服务层，Gateway 和 Nanopayments 更偏小额、低摩擦和 Agent 原生支付场景。

Circle 与 Coinbase 的差异

Coinbase 更像钱包、开发者和链上协议入口。Circle 更像稳定币发行、合规结算和机构网络。

在 Agent 支付里，Coinbase 可能更接近“谁让 Agent 发起支付”，Circle 更接近“支付最终用什么资产和网络清算”。

Circle 不一定控制 Agent 购物入口，但有机会控制 Agent 经济的稳定币结算层。
Cloud Distribution

八、AWS 与云平台：可能成为 Agent 支付的分发层

未来大量企业 Agent 不会单独运行在某个聊天界面里，而会部署在 AWS、Google Cloud、Microsoft Azure、Snowflake、Databricks、Salesforce、ServiceNow 等企业平台上。

一旦 Agent 在云平台上运行，它调用 API、工具、数据源、模型和外部服务时，都可能需要支付能力。

AWS Bedrock AgentCore Payments 把 Coinbase 和 Stripe 纳入其中，说明云平台正在尝试把支付能力变成 Agent 开发框架的一部分。

这意味着 Agent 支付的竞争不只发生在支付公司之间，也会发生在云平台、AI 平台和企业软件平台之间。

谁把支付能力嵌入 Agent 开发环境，谁就可能成为 Agent 支付的分发层。
Route Comparison

九、几套标准的本质区别

Stripe / OpenAI ACP

偏 AI 购物协议，解决 Agent 如何代表用户向商户下单。适合电商、品牌、平台、SaaS 商户和 AI App 内购买。

Coinbase x402

偏机器支付协议，解决 Agent 调用 API、数据、模型或工具时如何自动付费。适合微支付和按次计费。

Google AP2

偏授权与意图协议，解决用户如何授权 Agent 花钱，以及授权如何被验证、约束、撤销和审计。

Visa TAP

偏可信 Agent 身份协议，解决商户如何识别可信 Agent 和真实购买意图。

Mastercard Agent Pay

偏卡网络 Agent 支付基础设施，解决 Agent 如何在现有银行卡网络里安全付款。

PayPal 与 Circle

PayPal 偏钱包、商户保护和争议处理。Circle 偏稳定币结算资产、机构结算网络和微支付资金层。

Adoption Path

十、最终不会只剩一个标准

Agent 支付大概率是多标准并存。人类用户让 Agent 帮自己买商品，需要 ACP、AP2、Visa TAP、Mastercard Agent Pay 和 PayPal 钱包。Agent 调用 API 或 MCP 工具，需要 x402、稳定币钱包和纳米支付。银行和 PSP 做跨境后台结算，需要 Circle CPN、托管稳定币结算和合规网络。商户面对 AI 流量冲击，需要可信 Agent 身份、防机器人和风控协议。

未来更可能是：前端协议多样化，底层结算多轨并存，中间授权标准逐渐统一。

Investment Lens

十一、投资视角：谁最有机会吃到 Agent 支付红利？

1、Stripe：AI 商户支付编排层

Stripe 的优势是商户密度、开发者生态、支付产品完整度、Link 钱包和 OpenAI 合作。它最有机会成为 AI 购物场景中的商户支付编排层。

问题是 Stripe 不是上市公司，普通投资者难以直接买入。但它会影响 Shopify、OpenAI 生态商户、Google AI Mode 商户，以及 Visa、Mastercard、PayPal 的部分支付份额。

2、Coinbase：最纯的 Agent 原生支付协议押注

Coinbase 的优势是 x402、Base、USDC、Agent Wallets 和开发者生态。如果机器对机器支付、API 微支付、MCP 付费大规模增长，Coinbase 的战略位置会明显提升。

这对 COIN 的估值逻辑是重要升级。市场可能不再只把它当交易所，而是开始把它看成 Agent 钱包和机器支付协议公司。

3、Circle：Agent 支付最重要的稳定币结算资产之一

如果 Agent 支付最后大量使用稳定币结算，Circle 会直接受益。核心问题是 Circle 能否从 USDC 储备利息，升级到 CPN 交易费、托管服务费、微支付手续费和机构网络费。

这与 Circle 从稳定币发行商走向稳定币金融基础设施公司的逻辑一致。

4、Visa / Mastercard：不会轻易被稳定币绕开

Visa 和 Mastercard 仍然控制身份、授权、风控、tokenization、争议处理、全球商户接受网络和发卡收单关系。Agent 支付越普及，这些能力反而越重要。

中短期看，卡组织可能是 Agent 消费支付的受益者。长期风险在于稳定币和机器微支付可能侵蚀部分结算利润池。

5、PayPal：钱包入口和争议处理价值被重新放大

PayPal 在 Agent 购物中有机会重新强化钱包入口价值。用户更可能授权一个有买卖双方保护和争议处理能力的钱包给 Agent，而不是把原始银行卡暴露给未知 Agent。

它的关键挑战是能否从被动支付按钮升级为 Agent 钱包和商户服务层。

Risk Map

十二、需要持续验证的风险

标准碎片化

协议过多可能导致商户、钱包、云平台和 Agent 开发者重复适配。短期内，生态碎片化会降低采用效率。

责任边界不清

Agent 买错、重复执行、超出授权、订单争议、被恶意提示词诱导等问题，都需要明确用户、Agent 平台、钱包、商户和支付网络的责任划分。

监管差异

稳定币、消费者保护、数据权限、跨境资金流和身份认证，在不同司法区差异很大。Agent 支付越自动化，监管越会关注授权、审计和追责。

商业化验证

产品发布不等于收入放量。真正要看的指标是接入商户数量、真实交易量、take rate、客户留存、争议率、欺诈率、开发者采用率和企业集成深度。

Conclusion

十三、一句话总结

Stripe / OpenAI ACP 是 AI 购物协议。Coinbase x402 是机器微支付协议。Google AP2 是 Agent 授权协议。Visa TAP 是可信 Agent 身份协议。Mastercard Agent Pay 是卡网络 Agent 支付基础设施。PayPal 是钱包、商户保护和争议处理层。Circle 是稳定币结算网络和微支付资金层。

真正的大机会在于：Agent 从信息工具变成经济主体之后，谁负责授权、付款、结算、风控和责任归属。
