Core Thesis

Agent 支付不是单一协议竞争，而是授权、身份、钱包、结算和责任归属的系统性重构。

Stripe 有一套，Coinbase 有一套，但行业还包括 Google AP2、Visa TAP、Mastercard Agent Pay、PayPal 钱包层、Circle 稳定币结算层，以及 AWS 等云平台分发层。

AI 购物协议机器微支付Agent 授权委托可信身份稳定币结算
Framework

Agent 支付可以拆成五层

01

购物与下单协议

解决 AI Agent 如何代表用户向商户发现商品、确认订单、传递购物意图。

02

支付授权协议

解决用户如何授权 Agent 花钱，金额、商户、时间、任务范围如何被绑定。

03

钱包与资金账户

解决 Agent 使用银行卡、PayPal、Link、稳定币钱包还是企业资金账户。

04

结算网络

解决最终资金走卡网络、ACH、PayPal、USDC、PYUSD、银行间网络还是稳定币网络。

05

身份与责任归属

解决商户如何识别可信 Agent，支付出错、重复下单、欺诈和退款由谁承担。

AI

核心变化

Agent 从信息工具升级为经济执行主体，支付系统必须处理代理行为的可信性。

Standards Landscape

主要路线

S
Stripe / OpenAI ACP
偏 AI 购物协议。让 ChatGPT、AI App 与商户后端完成标准化下单和付款。
C
Coinbase x402
偏机器支付协议。适合 API、MCP server、数据源、模型调用和机器间结算。
G
Google AP2
偏授权与意图协议。核心是 mandate，即用户对 Agent 的支付授权如何被验证、约束、撤销和审计。
V
Visa TAP
偏可信 Agent 身份与商户风控。帮助商户区分真实购买代理和异常流量。
M
Mastercard Agent Pay
偏卡网络级 Agent 支付基础设施。通过 tokenization 和 Know Your Agent 把 Agent 纳入卡支付体系。
P
PayPal
偏钱包入口、商户网络和争议处理。把 PayPal 钱包、买卖双方保护和退款申诉接入 Agent 购物。
U
Circle USDC / CPN / Gateway
偏稳定币结算层。为 Agent 支付提供 24/7、跨境、可编程、低成本的资金清算资产和网络。
System View

未来不会只剩一个标准。

更可能是前端协议多样化，授权层逐渐标准化，底层结算多轨并存。消费者购物依赖卡网络、钱包和退款保护。机器对机器支付、API 调用、MCP 工具和纳米支付，更可能使用稳定币和链上结算。

Investment Lens

投顾视角：Agent 支付的四类资产

Agent 支付入口

代表：OpenAI、Stripe、Google、PayPal。核心看谁掌握用户和商户交互界面。

Agent 钱包与协议

代表：Coinbase、Stripe、PayPal、MetaMask、Privy、Bridge 相关生态。核心看谁能成为默认钱包和机器支付协议。

结算资产与网络

代表：Circle、Visa、Mastercard、PayPal、Coinbase。核心看稳定币、卡网络和钱包网络谁掌握最终清算层。

云平台分发层

代表：AWS、Google Cloud、Microsoft Azure。企业 Agent 会部署在云平台上，支付能力可能被嵌入开发框架。

Conclusion

一句话总结

Stripe / OpenAI ACP 是 AI 购物协议。Coinbase x402 是机器微支付协议。Google AP2 是 Agent 授权协议。Visa TAP 是可信 Agent 身份协议。Mastercard Agent Pay 是卡网络 Agent 支付基础设施。PayPal 是钱包、商户保护和争议处理层。Circle 是稳定币结算网络和微支付资金层。

真正的大机会在于：Agent 从信息工具变成经济主体之后，谁负责授权、付款、结算、风控和责任归属。
